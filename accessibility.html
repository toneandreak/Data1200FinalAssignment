Accessibility

Laying the Groundwork

We kept accessibility foremost in our thoughts from when we first started to build the website. We wanted to make sure that anyone would be able to use and enjoy it, which was why we not only focused on “passing” the accessibility checker, but also on other aspects of accessibility as well. We thought about the color choice and how it would impact readability for those with visual impairments, as well as language use for those with different reading disabilities, cognitive disabilities and learning difficulties. Instead of thinking along the lines of “catering” to those with disabilities, as it is often phrased, we have decided to build our website with the mindset that we are building without actively including barriers instead. Because of this, we decided that the Fun page should be accessible as well, not just the other pages.

Work and Results

In order to make sure our code was compatible with screen reading programs and that the pages themselves were technically accessible, we ran our website through the accessibility checker from Achecker at continously while we were working on it. This was to make sure that we wouldn’t have to go back and fix things when we were “done” but could make sure everything was done as correctly as possible from the get-go. In addition, we made most of our accessibility-related decisions beforehand, such as limiting our use of the div class, to make the website more accessible. We also chose to use a serif font, which is easier to read. The navigation bar can be tabbed through, and the _self tag is used as well. As we went along, certain problems cropped up, such as us using an h3 header under an h1 header, but these were the only two accessibility problems the accessibility checker showed that we had. We didn’t have any problems with color contrast, because we had chosen an accessible color scheme beforehand. When we had considered ourselves finished, we had 0 known problems and 0 likely problems. We managed to reduce our potential problems from 31 to 26 in accordance with the WCAG 2.0 (Level AA) guidelines.
